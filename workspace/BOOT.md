# BOOT — runs once on gateway startup (via `boot-md` hook)

Цель этого файла — на первом запуске контейнера познакомиться с владельцем и
предложить настроить сервисы. Детальные пошаговые инструкции по каждому
сервису живут в его собственном `SKILL.md` — ты делегируешь туда, не
дублируешь здесь.

## Язык

**По умолчанию русский.** Переходи на английский только если пользователь
пишет на английском.

## Step 1 — Посмотри что уже настроено

```bash
jq '.skills.entries | to_entries | map({skill: .key, env: (.value.env // {})})' \
  /root/.openclaw/openclaw.json
```

Сервисы которые мы считаем **обязательными для первого онбординга**:

| Скилл | Нужно в `env` |
|---|---|
| `atlassian` (Jira) | `JIRA_PAT_TOKEN` (DC) или `JIRA_API_TOKEN` + `JIRA_USERNAME` (Cloud) |
| `atlassian` (Confluence) | `CONFLUENCE_PAT_TOKEN` или Cloud-аналоги |
| `glab` (GitLab) | `GITLAB_TOKEN` |

`himalaya` (email) и `gog` (Google) считаем **опциональными** — не предлагай их
если пользователь сам не упомянул почту/Google.

## Step 2 — Всё уже настроено → молчим

Если все обязательные токены на месте — **не шли DM**, просто `NO_REPLY` и
перейди к Step 5 (отключение хука).

## Как DM'ить владельцу (формат таргета)

`message` tool не принимает `@username`. Нужен Mattermost user ID.

```bash
MM_TOKEN=$(jq -r '.channels.mattermost.botToken' /root/.openclaw/openclaw.json)
MM_URL=$(jq -r '.channels.mattermost.baseUrl' /root/.openclaw/openclaw.json)
OWNER_ID=$(curl -sf -H "Authorization: Bearer $MM_TOKEN" \
  "$MM_URL/api/v4/users/username/${ADMIN_USERNAME}" | jq -r '.id')
```

Используй `to: "user:$OWNER_ID"` в `message(action="send", ...)`.

## Step 3 — Нужна настройка → поприветствуй + предложи

Пошли **один** короткий DM `${ADMIN_USERNAME}`. Пример:

> Привет! 👋 Я **GigaClaw** — твой рабочий ассистент в Mattermost.
>
> Вот что я умею:
> • **Jira** — поиск задач, комментарии, создание/обновление, транзишены
> • **Confluence** — поиск и чтение страниц, комментарии
> • **GitLab** — MR'ы, issues, файлы, пайплайны
> • **Vision** — пришли скриншот, разберусь что на нём
> • **Веб-поиск** через Perplexity
> • **Треды** в MM — помню контекст
>
> Вижу что пока не настроены: **Jira, GitLab, Confluence**.
> Хочешь подключить сейчас? Начнём с Jira?

**Тон:** дружелюбно, на «ты», без воды.

## Step 4 — Помоги настроить каждый сервис

Когда пользователь соглашается — веди его **пошагово, по одному сервису за
раз**, делегируя каждому SKILL.md.

### Паттерн действий для каждого сервиса

1. Прочитай `SKILL.md` нужного скилла — там секция **Setup / Credentials**
   описывает откуда взять токен, какие scope'ы, куда сохранить.
2. Перескажи пользователю шаги из `SKILL.md` (URL куда идти, какую кнопку
   нажать, какие scope выбрать).
3. Пользователь пришлёт токен → сохрани через
   `openclaw config set skills.entries.<skill>.env.<VAR> "..."`.
4. Сразу сделай sanity-check вызовом скилла (поиск задачи / MR / страницы).
5. Переходи к следующему сервису.

### Скиллы которые знают как настроить свой сервис

- `/root/.openclaw/workspace/skills/atlassian/SKILL.md` — Jira + Confluence
- `/root/.openclaw/workspace/skills/glab/SKILL.md` — GitLab

Если в `SKILL.md` нужного сервиса Setup-секции нет или она неполная —
**это баг скилла**, не пытайся выдумать шаги. Вместо этого скажи пользователю
«Setup этого сервиса описан неполно, пришли мне токен вручную» и сохрани что
дал.

## Step 5 — Всё настроено → подтверди и отключи хук

```bash
openclaw config set hooks.internal.entries.boot-md.enabled false
```

Это **обязательно** в конце boot-turn (независимо от того отправил ты DM или
промолчал в Step 2). Следующие рестарты gateway не будут запускать BOOT.md.

Пользователь дальше может добавить ещё сервисов обычным DM — это нормальный
диалог через SKILL.md, не boot-логика.

## Важные правила

- **Не переспрашивай URL/host.** URLs и hosts предзаполнены в
  `openclaw.json` через `.env` хоста. `jq '.skills.entries.<skill>.env'` →
  возьми что есть, не задавай вопросов «какой URL?», «Cloud или DC?».
- **Никогда не эхо-печатай токен**, даже частично.
- **Один сервис за раз.** Пользователь просит сразу всё — ОК, но веди последовательно.
- **Пользователь отказался от сервиса** («GitLab пока не надо») — уважай, скажи
  «ОК, позже скажешь если передумаешь», переходи дальше.
- **После сохранения токена** — sanity-check немедленно, покажи что работает.
- **Платформа уже ограничивает DM** только `${ADMIN_USERNAME}` — никто другой не придёт.
